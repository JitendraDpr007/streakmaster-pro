## Goal

Transform questions from MCQ-only into 4 distinct types, each with the right solver UX. Existing MCQs stay as a "Quick Quiz" mode.

## Question types

| Type | Solver UX |
|---|---|
| `mcq` | Existing 4-option flow (unchanged) — "Quick Quiz" |
| `coding` | Story + problem → **Solve on LeetCode** / **Solve on GFG** buttons (open in new tab) + in-app **Notes editor** (approach, complexity, code scratch) saved per-user |
| `sql` | Monaco SQL editor + in-browser SQLite (sql.js). Show schema, seed rows, expected output. Run query → see actual rows → auto-grade vs expected |
| `system_design` | Tabbed walkthrough: Requirements → HLD → LLD → Tradeoffs. Each section is checklist-style. Floating "Discuss with AI Coach" button opens a chat drawer (Lovable AI Gateway, google/gemini-2.5-flash) |

## Schema changes (migration)

Add to `public.questions`:
- `type text not null default 'mcq'` — `'mcq' | 'coding' | 'sql' | 'system_design'`
- `leetcode_url text`, `gfg_url text` — coding
- `problem_statement text` — full problem (coding/sql)
- `sql_schema text`, `sql_seed text`, `sql_expected jsonb` — sql
- `hld jsonb`, `lld jsonb`, `requirements jsonb`, `tradeoffs jsonb` — system design (each is `[{title, points: string[]}]`)

New table `public.notes` for user coding notes:
- `id uuid pk`, `user_id uuid`, `question_id uuid`, `content text`, timestamps
- RLS: own-rows only, GRANTs to authenticated + service_role

Existing 85 MCQs keep `type='mcq'` automatically via default.

## Dependencies

```
bun add @monaco-editor/react monaco-editor sql.js
bun add -D @types/sql.js
```

sql.js wasm: lazy-load from CDN (`https://sql.js.org/dist/sql-wasm.wasm`) to avoid bundling 1MB into main chunk.

## AI Coach

`coach.functions.ts` uses `LOVABLE_API_KEY` → `google/gemini-2.5-flash`. Takes `{questionTitle, userMessage, history}`, returns assistant turn. System prompt makes it act as a senior staff engineer interviewer probing tradeoffs.
